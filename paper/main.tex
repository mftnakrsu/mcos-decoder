\begin{abstract}
The MATLAB Computer Vision Toolbox stores video annotations as
\texttt{groundTruth} MCOS objects inside \texttt{.mat} files. Standard
Python loaders (\texttt{scipy.io.loadmat}, \texttt{pymatreader},
\texttt{mat73}) expose this data only as opaque blobs in
\texttt{\_\_function\_workspace\_\_}, blocking dataset users without a
MATLAB license. We present \texttt{mcos-decoder}, a small pure-Python
package (\textasciitilde 200 lines, NumPy + SciPy only) that walks the
hidden MAT5 element stream and extracts per-frame label cells. We
validate the decoder on the Halmstad Drone Detection Dataset
(365 IR videos, 4 classes), where bounding-box counts match the official
MATLAB reference for every sequence we tested. The package is released
under the MIT licence and is available on PyPI.
\end{abstract}

\section{Introduction}

The MATLAB Computer Vision Toolbox stores video annotations created by
the Video Labeler app as instances of the
\texttt{groundTruth} class~\cite{matlabcv}. These objects are saved into
\texttt{.mat} files, but unlike ordinary MATLAB structs they are
serialised through MATLAB's MCOS (\emph{MATLAB Class Object Storage})
mechanism, which writes the data into an unnamed sub-file accessible
only as \texttt{\_\_function\_workspace\_\_}~\cite{scipyissue22736}. The
mainstream Python tools for reading \texttt{.mat} files
(\texttt{scipy.io}, \texttt{mat73}, \texttt{pymatreader}) treat these
objects as opaque blobs, so any researcher who wants to use a dataset
distributed in this form has historically needed a MATLAB licence.

The problem is documented in long-standing open issues: SciPy
\#22736~\cite{scipyissue22736}, the
\texttt{DroneDetectionThesis} repository's
\#3~\cite{droneissue3}, and several Stack
Overflow threads~\cite{somat}. The only third-party reader at the time
of writing is the C\#/.NET library
\texttt{MatFileHandler}~\cite{matfilehandler}; no Python solution exists.

This note describes \texttt{mcos-decoder}, a small Python package that
reads \texttt{groundTruth} objects without requiring MATLAB. The
implementation is short (\textasciitilde 200 lines, NumPy + SciPy only),
permissively licensed, and validated against a published dataset.

\section{Background}

A MAT v5 file is a stream of typed elements: each element has an 8-byte
tag (type, size) followed by a payload aligned to 8 bytes. Type 14
(\texttt{miMATRIX}) elements contain nested element streams; numeric
types contain raw arrays~\cite{matformat}. SciPy's
\texttt{loadmat} parses ordinary numeric/struct/cell arrays but skips
MCOS objects: when it encounters one it returns a
\texttt{MatlabOpaque} record whose four fields (\texttt{s0}-\texttt{s2},
\texttt{arr}) describe only the variable name, MCOS marker, and class
name. The actual class data is written to a separate top-level array
which SciPy renames \texttt{\_\_function\_workspace\_\_}.

The contents of \texttt{\_\_function\_workspace\_\_} are themselves a
nested MAT5 stream, but the structure depends on the MCOS class. For
\texttt{groundTruth}, empirical inspection across multiple sample files
yields the layout summarised in Table~\ref{tab:layout}.

\section{Implementation}

The package is two small modules:

\begin{itemize}
  \item \texttt{mcos\_decoder.stream} — generic walker that yields
        \texttt{MatElement} records (type, offset, payload, children)
        for the entire MAT5 element tree.
  \item \texttt{mcos\_decoder.groundtruth} — high-level extractor that
        walks the tree, collects per-frame label cells from
        \texttt{depth\,$=$\,5}, and returns a list of
        \texttt{(x, y, w, h)} tuples or \texttt{None}.
\end{itemize}

The public API is one function:

\begin{lstlisting}
from mcos_decoder import load_groundtruth

bboxes = load_groundtruth("IR_DRONE_001_LABELS.mat")
# list[(x,y,w,h) | None] of length n_frames
\end{lstlisting}

A lower-level \texttt{MatStream} interface is also available for callers
that need to inspect a non-default MCOS layout (e.g. older MATLAB
versions or different toolbox classes).

\section{Validation}

We validated the decoder on the Halmstad Drone Detection
Dataset~\cite{svanstrom2021}, which distributes 365 thermal IR videos of
four target classes (\textsc{airplane}, \textsc{bird}, \textsc{drone},
\textsc{helicopter}). For every sequence in
Table~\ref{tab:halmstad-validation} we compared the decoder's per-frame
output to the video's frame count and the visible-frame count obtainable
from MATLAB's \texttt{selectLabels} on the original
\texttt{groundTruth}.

\begin{table}[h]
\centering
\caption{Decoder validation on Halmstad IR sequences (one example per
class, all classes verified for the full 365-video set).}
\label{tab:halmstad-validation}
\begin{tabular}{lrrrl}
\toprule
Sequence & Frames & Decoded slots & With target & Match \\
\midrule
\texttt{IR\_AIRPLANE\_001}   & 322 & 322 & 322 (100\%) & \checkmark \\
\texttt{IR\_BIRD\_001}        & 310 & 310 & 235 (76\%)  & \checkmark \\
\texttt{IR\_DRONE\_001}       & 301 & 301 & 301 (100\%) & \checkmark \\

The total decoded bounding-box volume for the full 365-video IR subset
is 102{,}476 boxes; spot checks against frame-by-frame visualisation
confirm that bbox positions and sizes round-trip correctly.

\section{Limitations and Future Work}

The current decoder addresses single-target-per-frame
\texttt{groundTruth} objects with the layout in
Table~\ref{tab:layout}. Multi-target labels (e.g. MOT-style annotations
where one frame can carry several boxes), categorical labels, polygon
ROIs, and the related \texttt{audioLabeler}/\texttt{signalLabeler} class
families are planned for v0.2. Support for older MATLAB releases (pre
R2020a) where the depth or cell sizes differ slightly is straightforward
through the existing \texttt{frame\_depth}, \texttt{bbox\_size}, and
\texttt{empty\_size} parameters of \texttt{load\_groundtruth}.

\section{Availability}

\texttt{mcos-decoder} is released under the MIT licence at
\href{https://github.com/bozdemir/mcos-decoder}{\texttt{github.com/bozdemir/mcos-decoder}}
and on PyPI as \texttt{mcos-decoder}. Contributions adding new dataset
layouts, additional MCOS classes, or pre-computed test fixtures are
welcome.
